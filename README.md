# SongHelper

面向歌曲创作的 AI 辅助工具箱，也是我在本工作空间中的长期执行环境。

当前仓库目标：把“音频分析 → 人声/伴奏处理 → 旋律/和声提取 → 编曲辅助 → 混音参考 → 版本整理”等能力沉淀为一个可扩展工程，并在后续合作中持续积累、复用、迭代处理脚本与工具。

## 目前可做的事

- 用文本和推理协助创作：主题、情绪、歌词、段落结构、hook、和声建议、改编建议
- 用程序辅助分析：BPM、拍点、调性、和弦候选、段落切分、能量曲线、频谱观察
- 用工具链辅助音频处理：vocal/instrument 分离、片段裁剪、拼接、响度整理、效果链建议
- 为后续能力预留接口：旋律转 MIDI/简谱/五线谱、参考曲对比、项目版本归档、批处理流程

## 建议工作流

1. 把素材放入 `works/<作品名>/`
2. 在 `workspace/notes/` 记录创作意图、参考曲、风格目标
3. 用 CLI 查看支持能力：`python -m songhelper.cli capabilities`
4. 用 CLI 查看创作流程：`python -m songhelper.cli workflow`
5. 后续逐步接入具体音频处理脚本与第三方库

## 工程结构

- `docs/`：能力说明、创作流程、后续规划
- `src/songhelper/`：可扩展 Python 包与 CLI 入口
- `tests/`：基础测试
- `workspace/`：分析输出、乐谱、stems、mix、导出物
- `works/`：你的原始作品素材

## 工程定位

- 这是你的歌曲创作辅助仓库，也是我的执行环境
- 它的角色是工具箱，不替你接管创作流程、决策顺序或项目推进节奏
- 你后续交付给我的分析、编辑、转写、整理任务，都可以直接在这里沉淀为脚本、工具、配置和说明
- 这个仓库不是一次性 demo，而是一个会随着创作推进不断增强的“可复用音乐制作工具箱”
- 每次新增能力时，优先遵循：可复用、可验证、可批处理、可归档

详细定位见：`docs/PROJECT_POSITIONING.md`

## 后续建议接入的库

- 音频分析：`librosa`, `essentia`
- stem 分离：`demucs`, `spleeter`
- 人声/音高提取：`basic-pitch`, `crepe`, `parselmouth`
- 乐谱/MIDI：`music21`, `pretty_midi`, `mido`
- 音频编辑：`pydub`, `soundfile`, `pedalboard`
- 可视化与报告：`matplotlib`, `pandas`

> 当前初始化版本先提供工程骨架与能力清单，避免在未确认本机依赖前引入重型音频库。

## 协作约定

后续你可以直接给我如下类型任务：

- “分析这首歌的节奏、调性、和弦、结构”
- “把这段音频做人声/伴奏分离，并输出到指定目录”
- “从 vocal 提取旋律，生成 MIDI 或简谱草稿”
- “帮我给这段录音加一点混响 / 空间感建议”
- “把多个片段拼接，导出 demo”
- “为这类工作做成可重复执行的脚本工具，以后复用”

我会把当前工程当作工作空间和工具箱：

- 为具体任务创建或迭代脚本
- 把工具存放在仓库中便于复用
- 在 `docs/` 中沉淀工作流与使用说明
- 在 `workspace/` 中统一管理中间产物和导出结果
- 随着合作推进，逐步扩展为更完整的音乐创作辅助系统
- 但创作主流程仍由你决定，我只按你的任务指令提供能力与工具支持
